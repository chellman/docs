Design basics

Macaw allows for workflow that is similar to that of an image editor. You can draw and move elements freely on the canvas, then style them as you see fit. After that, if good HTML is important to you, you can organize and describe elements in the outline panel. The more Macaw knows about your elements purpose the better code output it will produce.

What's set's Macaw apart from other design software is it's ability to define elements dimensions both in pixels or percentages.

Canvas

When scrolling or panning the canvas vertically, all elements with fixed positioning remain in their places relative to the top and bottom canvas borders. Thanks to this smart feature you can see them just like they would behave in the browser.

Canvas in Macaw is fluid. At any moment you can grab the canvas resize handle on the right-hand-side and check how your design behaves when resized, and after you release the handle - canvas will spring back to its original size. You can permanently change canvas width through the breakpoint area in the Inspector Pane

In the design phase canvas height adjusts automatically, adding empty space at its bottom, allowing you to have maximum creative freedom. The Grid

Grid is an essential part of Macaw's workspace and workflow. It helps you to place elements evenly on the page and serves as a guide throughout the whole design process.

You can set up the grid to match your preferences, using Grid Width, Number of Grid Columns and Space Between Grid Columns. Like many other elements grid width can be set either as in percentages or pixel dimensions.

Drawing

Primary way of creating content in Macaw is by drawing. To draw some elements just pick the desired tool from the tool-bar then click + drag on the canvas. There are no constraints on where to draw an element, you can draw anywhere on the canvas, including on top of other elements. You can draw text elements, containers, buttons or even form elements. Images can be placed on the canvas right from within the library.

You can draw elements with percentage (%) or pixel (px) based widths and positions. These can be also adjusted later.

While drawing, elements will snap to grid, smart guides, canvas boundaries or other elements.

Elements

Elements can be drawn with the Element Tool R, when converted to code they will become block level elements such as divs, sections, forms, etc.

Containers

Design wise containers are used for grouping other elements, but unlike groups in standard design software, containers have their own dimensions (or even paddings) and can be styled just like standard elements.

You can create containers by using the Elements > Group command. Another way of creating containers is by using the Container Tool G, with it selected simply drag over some elements to wrap them in a container. Groups created with Container Tool will immediately have dimensions based on the area you've dragged over while drawing, whereas grouping with Elements > Group command will adjust the group size to fit it's children.

If among the elements that you are grouping together with Elements > Group command, one seems to fit the task of containing all the other elements, Macaw will ask you if you want to transfer presentation attributes of that element to the newly created group. In other words: Macaw is smart enough to use existing elements as groups when it makes sense, and leave you with cleaner code in the process.

Text

Macaw has the advantage of using a browser based canvas, which allows the text to be rendered exactly like it will be in the target browser.

Paragraph and Point text fields

You can create two types of text boxes in Macaw. Simply clicking on the canvas with the Text Tool selected, will create a Point Text field that has auto width and behaves like an inline element.

To create a block based text box, that has set dimensions and behaves like a block level element, click and drag on the canvas with Text Tool selected.

You can also toggle between two types of text boxes after you've created a field, by choosing an adequate value from width field's units in Dimensions section of Inspector Pane. Choosing Auto will set the text box to Point Text, whereas px or % will set the text box to Paragraph Text.

Text field dimensions

When a text element is selected, resize handles help you identify it's type and properties. If side handles are present, they indicate that element's width is set in px or %. Top and bottom handles indicate that this text element has set height in px or %. Lack of handle pair means that this property is set to auto.

Paragraphs versus line breaks

Hitting Return/Enter to separate lines of text inside a text box will result in code output for that box being divided into several paragraphs. You can also use Shift + Return/Enter to create line breaks (<br>) in the code output.

Fast text prototyping

Macaw supports Lorem Ipsum text generation, for quickly filling text boxes with sample copy. To add generated text to a text box double click the text box, type loremXp (where X stands for number of paragraphs you wish to create) then hit TAB and Macaw will generate X paragraphs of Lorem Ipsum….

For finer amounts of generated text, you can type loremXw instead (where X stands for number of words you wish to add) then hit Tab and Macaw will add X words to that text box.

Both shortcuts can be used multiple times in one text box, to fill it with sample content just as you see fit.

Inputs

Buttons

Buttons are drawn using the button tool B. After clicking & dragging on the canvas, the button stays in edit mode, where you can set it's text. Button text is centered by default.

If you attach a URL to a button using Link field inside Inspector Pane, the button will redirect to that URL through an onClick JavaScript event. To change the button to a standard link, change the button tag name inside Inspector Pane to a, it will be then converted to a plain <a> tag in the output.

Input text

After drawing an input field, you can type some text inside, that will be used as a HTML5 placeholder text for that field.

For Input text field, in the Inspector Pane, apart from element tag name you can additionally set it's name attribute. You can also choose input type, from various types supported by HTML 5, e.g.: text, email, password, number, search, URL.

Text area

Just like with Input text, after drawing a Text Area you can set it's text which will be used for HTML5 placeholder text.

In the Inspector Pane you can set Text area's name attribute and set how it should react to resizing attempts after publishing in the browser.

Select

Like other form elements, the Select Element can also have it's HTML name attribute set.

Select supports multiple option elements, you can add, reorder and remove them from within Inspector Pane. First top-most option will be used as the default one.

Checkbox & radio

Checkbox and radio elements can be drawn by clicking and dragging on the canvas with their respective tools. Both elements have a minimum width of 66 px and minimum height of 22 px, that are applied when drawing, but can be adjusted later.

Positioning

Positioning types

Macaw supports three positioning types: static (default), absolute and fixed. For every element, you can pick which positioning should be used by Macaw.

To make it easier to identify element positions, element selection boxes, margin indicators and even part of UI that controls the position have colors based on selected element position type. This allows for easy inspection whenever you select an element.

Following colors are used for marking position type:

  • Static - Blue,
  • Absolute - Orange,
  • Fixed - Green.

Static positioning is the default used by Macaw. Elements with static position are placed using their margin values relating to the canvas or other elements. If elements overlap - negative margins may be used.

When static positioning is set to a bunch of elements - they might move if you move the element that their margin relates to. (Really unsure if this is a bug or a feature - needs to be verified. Also might need an animation)

Absolute positioning allows the elements to be independent of other elements positions. It can also be used to position elements inside their parents.

Fixed positioning is useful for elements that have to be fixed (pinned) to the page and stay at their position even if the page is scrolled. This allows you to see their behavior during design phase, just like you would in the browser.

Origins

Origins in Macaw define how an element is placed on the canvas in relation to other elements. It directly translates to how that element's margins are set. For statically positioned elements you have the option to set X origins, whereas absolutely and fixed positioned elements additionally allow you to set Y origins.

X origin can be set to left, right or center. X center origin means that an element will be always centered inside it's parent, left and right origins will allow an element to be positioned relatively to the desired side of it's parent.

Y origin can be set to top, bottom or middle. Y middle origin means that an element will be always centered vertically inside it's parent. By combining X center and Y middle origins you can place an element exactly in the middle of it's parent. Top and bottom Y origins position the element in the desired distance either from top or bottom of it's container.

By using fixed positioning you can easily create sticky headers and footers just by adjusting elements position and setting Y origin to top or bottom.

Margins

Margin indicators (solid semi-transparent blocks that show up on element's sides when it's selected) help to determine how an element will be placed on the canvas with relation to the surrounding elements.
